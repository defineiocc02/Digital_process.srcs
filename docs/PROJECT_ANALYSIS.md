# SAR ADC 数字处理工程 — 技术分析与实现指南

## 一、工程概述

### 1.1 系统定位

本工程实现了**SAR ADC的数字后端处理系统**，包含两条核心功能主线：

| 功能主线 | 描述 | 对应模块 | 实现文件 |
|---------|------|---------|----------|
| **校准 (Calibration)** | 在已知条件下测量电容权重，写入存储，用于后续转换的加权计算 | 校准控制器 + 虚拟ADC物理模型 | `sar_calib_ctrl_serial.sv` + `virtual_adc_phy.v` |
| **重构 (Reconstruction)** | 使用已校准的权重对SAR原始比特(raw bits)进行加权求和，得到最终数字码输出 | 重构引擎 | `sar_reconstruction.sv` |

**核心关系**：
- **校准** 产出：权重 `w[0..CAP_NUM-1]`（通过 `w_wr_en / w_wr_addr / w_wr_data` 写入）
- **重构** 使用：同一套权重 + 每次转换的 `raw_bits` → 输出 `adc_dout`（16-bit 有符号）

**工作模式**：校准是离线/上电一次（或周期重校），重构是每拍/每样本都在执行。

## 二、核心模块实现分析

### 2.1 校准控制器 (sar_calib_ctrl_serial)

#### 2.1.1 功能与特性

- **核心功能**：实现"Measure-then-Set"递归校准算法，测量各比特权重并更新到阴影RAM
- **关键特性**：
  - 递归测量：从低位到高位依次校准
  - 串行累加：优化权重计算，减少时序瓶颈
  - 偏移消除：使用(P+N)/2方法消除偏移
  - MSB保护：防止高位校准溢出
  - ASIC安全初始化：复位时初始化默认权重

#### 2.1.2 工作流程

1. **初始化**：设置目标校准比特，初始化累加器和计数器
2. **P相测量**：
   - 设置P相输入（dac_p_force[target_bit] = 1）
   - 执行SAR搜索，找到平衡码
   - 串行计算P相权重值
3. **N相测量**：
   - 设置N相输入（dac_n_force[target_bit] = 1）
   - 执行SAR搜索，找到平衡码
   - 串行计算N相权重值
4. **累加与平均**：累加P相和N相测量值，重复AVG_LOOPS次求平均
5. **更新权重**：计算平均值并写入外部接口和阴影RAM
6. **迭代**：目标比特递增，重复上述过程直到所有比特校准完成

#### 2.1.3 关键实现细节

- **状态机设计**：10个状态，包括空闲、初始化、P相/N相的设置/搜索/计算、累加、更新权重和完成
- **串行累加**：使用temp_acc寄存器逐位累加权重，减少并行加法器的时序压力
- **MSB保护**：对于Bit 18/19，额外添加低位权重以防止溢出
- **权重计算**：(accumulator + 0.5) >> (AVG_SHIFT + 1)，实现四舍五入

### 2.2 重构模块 (sar_reconstruction)

#### 2.2.1 功能与特性

- **核心功能**：使用校准权重对SAR原始比特进行加权求和，输出16位有符号数字码
- **关键特性**：
  - 流水线设计：分为部分累加(Stage 1)和全局累加(Stage 2)
  - 40位动态范围：防止中间结果溢出
  - 0.5 LSB偏移补偿：提高输出精度
  - 动态权重更新：支持实时校准

#### 2.2.2 工作流程

1. **权重存储**：使用本地RAM存储各比特权重，支持动态更新
2. **部分累加**：将20个输入分为4组，每组5个输入，计算各组部分和
3. **全局累加**：将4组部分和相加得到总和
4. **缩放与饱和**：
   - 除以2（因为输入范围是2*Vref）
   - 加0.5 LSB进行舍入
   - 右移FRAC_BITS位进行缩放
   - 饱和处理，确保输出在16位有符号范围内

#### 2.2.3 关键实现细节

- **流水线设计**：两级流水线，提高吞吐量
- **部分累加**：20 inputs / 4 groups = 5 inputs per group，平衡资源与性能
- **缩放计算**：TOTAL_SHIFT = FRAC_BITS，确保权重格式与输出格式匹配
- **符号扩展**：使用signed'()进行符号扩展，确保计算正确性

### 2.3 虚拟ADC物理模型 (virtual_adc_phy)

- **功能**：使用dac_p_force和dac_n_force作为等效DAC输出，产生比较结果comp_out
- **应用场景**：
  - FPGA上闭环验证校准算法
  - 配合ILA进行调试
  - 无需真实ADC模拟前端即可验证校准逻辑

### 2.4 FPGA顶层包装器 (fpga_top_wrapper)

- **功能**：集成校准控制器和虚拟ADC物理模型，提供FPGA验证接口
- **接口**：
  - 输入：clk, rst_n_btn, start_sw
  - 输出：done_led
- **特点**：当前版本只包含校准逻辑，未集成重构模块

## 三、技术参数与配置

| 参数 | 值 | 说明 | 应用模块 |
|------|-----|------|----------|
| CAP_NUM | 20 | 电容总位数 | 校准 + 重构 |
| WEIGHT_WIDTH | 30 | 权重存储位数（有符号） | 校准 + 重构 |
| OUTPUT_WIDTH | 16 | 输出数据位数 | 重构 |
| FRAC_BITS | 8 | 权重小数部分位数（Q22.8格式） | 重构 |
| AVG_LOOPS | 32 | 平均次数 | 校准 |
| COMP_WAIT_CYC | 16 | 比较器/DAC稳定等待周期 | 校准 |
| MAX_CALIB_BIT | 5 | 预校准LSB最高位 | 校准 |
| TOTAL_SHIFT | FRAC_BITS | 总移位数 | 重构 |

## 四、测试与验证

### 4.1 重构模块测试 (tb_sar_recon)

- **测试内容**：
  - 线性度测试：验证输入电压与输出码的线性关系
  - 权重更新灵敏度测试：验证权重变化对输出的影响
  - 流水线吞吐测试：验证连续输入时的处理能力

- **测试方法**：
  - 使用`generate_ideal_bits(vin)`将归一化电压(-1~+1)转换为20-bit理想码
  - 使用`force_ideal_weights()`灌入理想权重W[i] ∝ 2^(i+4)
  - 验证输出与理论值的偏差

### 4.2 校准模块测试

- **FPGA验证**：在FPGA上运行fpga_top_wrapper，通过ILA观察：
  - 校准状态机的转换
  - 权重计算过程
  - 最终权重值
  - 校准完成信号

## 五、问题分析与改进建议

### 5.1 架构/集成问题

| 问题 | 影响 | 建议解决方案 |
|------|------|-------------|
| 校准与重构在顶层未集成 | 无法在同一FPGA映像中完成校准+采样的完整流程 | **方案A**（推荐）：设计完整FPGA顶层，包含校准和重构，支持模式切换<br>**方案B**（最小改动）：保持现有结构，通过JTAG/寄存器预置权重 |
| testbench与FPGA顶层不对应 | 缺少校准+重构的联合测试 | 创建联合测试平台，验证校准后的权重在重构中的表现 |

### 5.2 实现/维护问题

| 问题 | 影响 | 建议解决方案 |
|------|------|-------------|
| 复位未消抖 | 按键抖动可能导致校准异常 | 添加消抖电路，或明确说明由外部硬件/固件消抖 |
| 理想权重与参数易不同步 | RTL参数改动时，测试可能失败 | 在共享头文件中定义统一参数，TB和RTL都引用 |
| 线性度判据过松 | 无法准确评估性能 | 记录adc_dout - ideal_expect，可选输出到文件或做直方图 |
| 流水线背压测试不完善 | 无法验证连续输入时的正确性 | 在TB中对DUT的ready/valid握手建模 |
| 逻辑优化问题 | 关键逻辑可能被综合器优化掉 | 保留mark_debug，必要时使用keep/dont_touch属性 |

### 5.3 性能优化建议

1. 校准速度优化：
   - 考虑并行校准方法，减少校准时间
   - 优化状态机转换，减少等待周期

2. 重构性能优化：
   - 根据目标FPGA特性，调整流水线深度
   - 考虑使用DSP48单元进行权重乘法，提高计算效率

3. 资源利用优化：
   - 权重RAM共享：校准和重构使用同一权重RAM，减少资源占用
   - 逻辑复用：校准和重构中的公共逻辑可以复用

## 六、使用指南

### 6.1 FPGA验证流程

1. 校准模式：
   - 启动FPGA，按下复位按钮
   - 打开启动开关(start_sw)，开始校准
   - 等待完成指示灯(done_led)亮起，表示校准完成

2. 正常模式（需要修改顶层）：
   - 提供真实ADC的raw_bits输入
   - 重构模块使用已校准的权重计算输出
   - 观察adc_dout输出

### 6.2 测试方法

1. **重构模块测试**：
   ```bash
   cd Digital_process.sim/sim_1/behav/xsim
   ./simulate.bat tb_sar_recon
   ```

2. **校准模块测试**：
   - 在Vivado中打开工程
   - 运行Implementation
   - 下载比特流到FPGA
   - 使用ILA观察校准过程

### 6.3 参数配置

- 校准参数：在sar_calib_ctrl_serial.sv中修改：
  - AVG_LOOPS：平均次数，影响校准精度和时间
  - COMP_WAIT_CYC：等待周期，影响校准速度

- 重构参数：在sar_reconstruction.sv中修改：
  - FRAC_BITS：小数位数，影响权重精度
  - OUTPUT_WIDTH：输出位数，根据需求调整
